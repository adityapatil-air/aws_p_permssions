ShipFile Documentation

Getting Started

ShipFile transforms your AWS S3 bucket into a powerful file sharing platform. Follow this guide to set up your account and start sharing files securely.

Prerequisites: You need an AWS account with S3 access to use ShipFile.

AWS Setup Guide

Before using ShipFile, you need to configure your AWS account properly. Follow these steps:

1. Create S3 Bucket

  1. Log in to your AWS Console
  2. Navigate to S3 service
  3. Click "Create bucket"
  4. Choose a unique bucket name (e.g., "my-company-shipfile")
  5. Select your preferred AWS region
  6. Keep default settings for now
  7. Click "Create bucket"

2. IAM User & Group Setup

Create a dedicated IAM user for ShipFile with appropriate permissions:

Create IAM Group

  1. Go to IAM service in AWS Console
  2. Click "User groups" → "Create group"
  3. Group name: "ShipFileUsers"
  4. Attach the following policies:
    • AmazonS3FullAccess
  5. Click "Create group"

Create IAM User

  1. Go to "Users" → "Create user"
  2. Username: "shipfile-user"
  3. Select "Programmatic access"
  4. Add user to "ShipFileUsers" group
  5. Click "Create user"
  6. Important: Save the Access Key ID and Secret Access Key

Security Note: Store your AWS credentials securely. Never share them or commit them to version control.

3. Configure CORS

Set up Cross-Origin Resource Sharing (CORS) for your S3 bucket:

  1. Go to your S3 bucket
  2. Click "Permissions" tab
  3. Scroll to "Cross-origin resource sharing (CORS)"
  4. Click "Edit" and paste the following configuration:
[
    {
        "AllowedHeaders": ["*"],
        "AllowedMethods": ["GET", "PUT", "POST", "DELETE", "HEAD"],
        "AllowedOrigins": ["*"],
        "ExposeHeaders": ["ETag"],
        "MaxAgeSeconds": 3000
    }
]
  1. Click "Save changes"

Permission System

ShipFile offers granular permission control for different user roles:

View Permissions

  • • View Own: See only uploaded files
  • • View All: See all files in bucket

Upload Permissions

  • • Upload + Manage Own: Upload and manage own files
  • • Upload + Manage All: Upload and manage all files

Extra Permissions

  • • Download Files: Download files from the bucket
  • • Generate Share Links: Create time-limited share links
  • • Create Folders: Create new folders
  • • Invite Members: Invite new team members

Team Management

Creating Organizations

  1. After connecting your S3 bucket, click "Create Organization"
  2. Enter your organization name
  3. Click "Create Organization"

Inviting Members

  1. Click "Invite Member" button
  2. Enter member's email address
  3. Configure permissions and access scope
  4. Choose between "Entire Bucket" or "Specific Folders"
  5. Send invitation

File Sharing

Creating Share Links

  1. Select files or folders to share
  2. Click "Share" button
  3. Choose expiration time (1 hour to 30 days)
  4. Click "Generate Link"
  5. Copy and share the generated URL

Tip: Share links automatically expire for security. Recipients don't need accounts to access shared files.

Troubleshooting

Common Issues

Invalid AWS Credentials

Error: "Invalid AWS credentials" or "Access Denied"

Solution: Verify your Access Key ID and Secret Access Key. Ensure the IAM user has S3 permissions.

CORS Errors

Error: "CORS policy" or "Cross-origin request blocked"

Solution: Configure CORS settings in your S3 bucket as described in the setup guide.

Upload Failures

Error: "Upload failed" or "Permission denied"

Solution: Check user permissions and ensure you're in the correct folder if you have limited access.
